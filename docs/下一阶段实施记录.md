## 新增 API 基座

| API | 用途 |
| --- | --- |
| `GET /api/v1/admin/clusters` | 查询 OpenSandbox 集群 |
| `POST /api/v1/admin/clusters` | 注册 OpenSandbox 集群 |
| `PUT /api/v1/admin/clusters/{cluster_id}` | 创建或更新指定 OpenSandbox 集群 |
| `GET /api/v1/admin/ack-deployments` | 查询 ACK/K8s 接入和 OpenSandbox 套件部署记录 |
| `POST /api/v1/admin/ack-deployments` | 创建 ACK/K8s 接入部署记录 |
| `POST /api/v1/admin/ack-deployments/{deployment_id}/plan` | 生成 OpenSandbox 套件 Kubernetes manifest 部署计划 |
| `GET /api/v1/admin/images` | 查询 OpenSandbox 沙箱镜像目录 |
| `POST /api/v1/admin/images` | 登记手动上传或外部镜像源 |
| `POST /api/v1/admin/images:upload` | 上传镜像文件，保存本地制品并自动生成到各集群的分发计划 |
| `GET /api/v1/admin/image-distributions` | 查询镜像到各集群的分发状态 |
| `POST /api/v1/admin/images/{image_id}/distributions:sync` | 为镜像生成到所有启用集群的分发计划 |

## 当前边界

本批暂不做真实外部资源联调：

- 不直接调用阿里云 ACK OpenAPI。
- 不保存真实 kubeconfig 内容，只记录 `kubeconfig_secret_ref`。
- 暂不实际推送镜像到远端仓库；当前先保存上传制品并生成分发计划和状态模型。
- 不执行 Helm/Kustomize/manifest apply，只记录部署意图和状态。

## 安全基线测试增强

本批继续推进近期任务清单中的安全测试覆盖，补齐生产验收中必须长期保持的失败路径：

| 场景 | 覆盖方式 |
| --- | --- |
| 禁用的云沙箱凭据不可用 | 服务层测试校验 `disabled` key 返回 `INVALID_CLOUD_SANDBOX_CREDENTIAL` |
| 过期的云沙箱凭据不可用 | 服务层测试校验过期 key 返回 `INVALID_CLOUD_SANDBOX_CREDENTIAL` |
| 禁用用户不可继续使用 key | 服务层测试校验用户状态非 active 时返回 `FORBIDDEN` |
| 无 OpenSandbox Plus 角色用户不可使用 key | 服务层测试校验缺少 `osb_agent_user`/`osb_platform_admin` 时返回 `FORBIDDEN` |
| 跨用户沙箱访问不可见 | 服务层测试确认沙箱查询始终带 owner subject，跨用户访问返回 `NOT_FOUND` |
| OIDC audience/issuer 错误必须拒绝 | Token verifier 测试确认错误 audience/issuer 被包装为 `TokenValidationError` |

当前覆盖重点是本地可重复验证的安全策略。真实企业 OIDC、ACK/K8s 与镜像仓库联调阶段，还需要结合实际 Casdoor/企业 IdP 配置补充端到端错误 token、禁用用户同步和权限映射测试。

## 结构化日志与 request_id 透传

本批继续推进近期任务清单中的可观测基础能力：

| 能力 | 当前落地 |
| --- | --- |
| 结构化访问日志 | 所有请求完成后输出 JSON 结构化日志，包含 `request_id`、method、path、status、duration、user_id、credential_id、client_ip 和未处理异常标记 |
| 管理面用户关联 | 管理 API 认证通过后把当前 principal 写入 request state，访问日志可记录 `user_id` |
| 兼容 API 凭据关联 | OpenSandbox 兼容 API 校验 key 后把 cloud sandbox principal 写入 request state，访问日志可记录 `credential_id` |
| 下游 OpenSandbox 透传 | 普通 OpenSandbox client 调用和 proxy 转发都会向下游写入 `X-Request-ID` |
| 内网调用稳定性 | OpenSandbox client/proxy 使用 `trust_env=False`，避免宿主机代理变量影响容器内网服务调用 |
| 回归测试 | 新增访问日志字段测试和 OpenSandbox client request_id 透传测试 |

当前尚未引入 Prometheus metrics、Trace 和告警系统；这些会在可观测阶段结合 dashboard 和真实部署环境继续补齐。
